# BUSINESSNow v7 — End-to-End Testing Prompt
## Coherence · Completeness · Quality

**Platform:** KSAP RPM / BUSINESSNow v7  
**Scope:** Full system test — all modules, all roles, all data flows  
**Excludes:** QuickBooks Online integration (internal accounting module used instead)  
**Accounting Model:** Internal GL ledger with manual payment tracking (see Section 8)  
**Prerequisites:** Production database seeded, real auth enabled, all Phase 1–5 builds complete

---

## How to Use This Document

Each section is a self-contained test block. Work through them **in order** — later sections depend on data created in earlier ones. Each test includes:

- **Setup** — data or state required before running the test
- **Steps** — exact actions to perform in the UI or via API
- **Pass Criteria** — specific, observable outcomes that confirm the feature works
- **Fail Signals** — what to watch for that indicates a problem

Do not skip sections. A pass in Section 6 (Invoicing) is meaningless if Section 3 (Time Entry) data is not correctly approved.

## Section 2 — Project Execution & WBS

### Test 2.1 — Phase Gate Entry Criteria

**Setup:** ACME-001 Phase 1 (Discovery) is in "Not Started" status. Entry criteria have been configured: `["Kickoff meeting completed", "NDA signed", "Access to client systems confirmed"]`.

**Steps:**
1. Log in as Morgan PM.
2. Navigate to ACME-001 → Phases.
3. Attempt to move Phase 1 from "Not Started" to "In Progress".
4. Observe what appears.
5. Check only the first two criteria. Attempt to save.
6. Check all three criteria. Attempt to save.

**Pass Criteria:**
- Step 4: A gate checklist modal appears listing all three entry criteria as unchecked checkboxes. The "Proceed" or "Confirm" button is disabled.
- Step 5: With only two of three checked, the button remains disabled or a validation message appears preventing the transition.
- Step 6: With all three checked, the button becomes active. Clicking it transitions Phase 1 to "In Progress". The criteria are stored with the checking user's name and timestamp.

**Fail Signals:**
- Phase status changes without any checklist appearing.
- Checklist appears but allows saving with unchecked items.
- Criteria are not persisted — refreshing the page clears which criteria were checked.

### Test 2.2 — Task Dependencies & Gantt Cascade

**Setup:** ACME-001 Task 1.1 (Kickoff) has a Finish-to-Start dependency on Task 1.2 (Current State Assessment). Task 1.1 end date is set to Day 5. Task 1.2 start date is Day 6.

**Steps:**
1. Log in as Morgan PM.
2. Open ACME-001 → Gantt view.
3. Confirm a dependency arrow is drawn from Task 1.1's right edge to Task 1.2's left edge.
4. Drag Task 1.1's end date to extend it by 3 days (now ends Day 8).
5. Observe Task 1.2's position.

**Pass Criteria:**
- Step 3: A visible arrow connects Task 1.1 to Task 1.2 on the Gantt chart.
- Step 5: Task 1.2 automatically shifts its start date to Day 9 (immediately after Task 1.1's new end). The cascade happens without manual intervention.
- If Task 2.1 also has a dependency on Task 1.2, it cascades further in the same action.
- The updated dates are persisted in the database (confirm by refreshing the page).

**Fail Signals:**
- No dependency arrows visible on the Gantt.
- Dragging Task 1.1 does not move Task 1.2.
- Task 1.2 moves visually but reverts to original date on page refresh.

### Test 2.3 — Project Baseline & Variance View

**Setup:** ACME-001 with tasks and estimated hours as seeded. No baseline has been taken yet.

**Steps:**

2. Navigate to ACME-001 header. Find the "Baseline This Project" action.

**Pass Criteria:**

- Step 6: The Variance tab renders a table with columns: Task | Baseline Hours | Actual Hours | ETC Hours | Variance.
- Task 1.1 shows: Baseline = 16h, Actual = 0h (no time logged yet), ETC = 20h, Variance = +4h over baseline.

**Fail Signals:**

- Variance tab is absent from Project Detail.

### Test 2.4 — Project Closure Gate

**Setup:** ACME-001 with approved time entries, actual completion date and performance rating not yet set.

**Steps:**
1. Log in as Morgan PM.
2. On ACME-001, attempt to change project status to "Completed" directly.
3. If a gate modal appears, do not fill in any fields. Attempt to submit.
4. Fill in: Actual Completion Date = today's date. Performance Rating = 4 stars. Confirm all time is approved.
5. Submit.

**Pass Criteria:**
- Step 2: The system does not immediately change the status. A closure gate wizard or modal appears.
- Step 3: The wizard blocks submission. It shows a checklist: (1) All time approved — must show ✓ or ✗ based on actual state, (2) Actual Completion Date — required field, (3) Performance Rating — required 1–5 selector.
- Step 5: With all gate conditions satisfied, the project status transitions to "Completed". A profitability snapshot is frozen at this moment (margin % is locked in project record).
- If any approved time is missing, the gate clearly identifies which resource/period still has pending time before allowing closure.

**Fail Signals:**
- Status changes to "Completed" without any gate.
- Gate appears but can be submitted without the performance rating.
- Profitability snapshot is not captured on closure (margin continues to update after project is "Completed").

**Setup:** Logged in as Sam Consultant. ACME-001 is assigned.
