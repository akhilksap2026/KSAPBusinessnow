─── 1. API: PROSPECTS ENDPOINTS ───
Create api-server/src/routes/prospects.ts:

GET /api/prospects?status=&ownerId=
  Requires role: account_manager | delivery_director | admin.
  Return 403 for all other roles.
  Returns all prospects.
  IMPORTANT: strip confidential fields for any role that is not
  account_manager, delivery_director, or admin:
    delete prospect.primaryContactName
    delete prospect.primaryContactEmail
    delete prospect.linkedinUrl
    delete prospect.sentiment
    delete prospect.touchPoints

GET /api/prospects/:id
  Same field-level security as above.
  Also includes linked opportunities (WHERE prospectId = :id).

POST /api/prospects
  Body: { name, type, industry, segment, primaryContactName,
          primaryContactEmail, linkedinUrl, sentiment, ownerId, notes }

PUT /api/prospects/:id
  Full update.

DELETE /api/prospects/:id
  Only if status != 'converted'.

POST /api/prospects/:id/convert
  This operation MUST be transactional:
  1. BEGIN TRANSACTION
  2. Read the prospect record
  3. INSERT into accounts:
     { name: prospect.name, type: prospect.type, industry: prospect.industry,
       segment: prospect.segment, status: 'active',
       convertedFromProspectId: prospect.id }
  4. UPDATE prospects SET
       convertedToAccountId = [new account id],
       convertedAt = NOW(),
       status = 'converted'
     WHERE id = :id
  5. COMMIT
  6. If any step fails: ROLLBACK and return 500
  Returns: { accountId: [new account id] }
  Frontend then navigates to /customers/:accountId.

Register in routes/index.ts.
